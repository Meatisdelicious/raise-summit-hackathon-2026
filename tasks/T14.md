# T14 — Inference adapter (live / replay / recording)

Implement the `InferenceClient` protocol (`docs/CONTRACTS.md` §4) in all modes (spec §8.2/§18).
**Vultr stays the live critical path**; replay/stub are for CI/tests only.

## Do
- `vultr.py` — `VultrInferenceClient`: OpenAI-compatible calls to `VULTR_INFERENCE_BASE_URL`/
  `VULTR_MODEL`, temperature 0, request timeouts + bounded retries; every response validated against the
  schemas; on invalid structure, retry once with the validation error.
- `replay.py` — `ReplayInferenceClient`: hash each request (model + prompt + tool-result context) →
  committed cassette in `apps/api/tests/cassettes/`; deterministic.
- `recording.py` — `RecordingInferenceClient`: wraps Vultr, writes cassettes (`make record`).
- `factory.py` — select client by `LOOPCLOSER_INFERENCE_MODE` (`live`/`replay`/`stub`).
- Unit tests for the factory + structured-output validation/retry (using stub + a tiny fake cassette).

## DoD
- [ ] Factory + validation/retry unit-tested. [ ] Replay deterministic. [ ] No secrets logged.
- [ ] `make verify` green. [ ] Diff within `owned_paths`. [ ] PR opened.
