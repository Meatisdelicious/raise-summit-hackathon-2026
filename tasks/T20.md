# T20 — Retrieval + object storage + ingestion

Spec §9 (retrieval), §8.2 (storage). Storage adapter is swappable: local/minio for dev + Vultr Object
Storage for prod (private bucket, short-lived signed URLs).

## Do
- `storage/`: S3-compatible client (mock/local + Vultr) + ingestion that extracts page-aware text
  (PyMuPDF/pypdf), chunks it, and persists `documents` + `document_chunks` with `{document_id, page,
  start_offset, end_offset}` citations.
- `retrieval/`: `search_documents` (filter by case + document_type + date range, exact alias match,
  then semantic rank; `limit`) and `get_document_context` (cited page + adjacent context). Backs the
  tool implementations in T21.
- Integration test: for the signature case E, retrieval surfaces the tempting-but-invalid
  (predates-instruction) candidate **and** the correct absence — so the agent can reject correctly.
