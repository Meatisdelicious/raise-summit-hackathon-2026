# T12 — Deterministic validators + decision state machine (no LLM)

The safety-critical core (spec §6.4/§6.5/§13). **The LLM never touches this** — it must decide A–H
purely from pre-extracted fixtures.

## Do
- `policies/`: deterministic validators — deadline resolution (relative + explicit; ambiguous →
  needs_review), target/alias normalization (versioned table), temporal validity (evidence after
  instruction date), final-vs-draft status, appointment-vs-completion distinction, same-case check,
  operational window, documented-exception acceptance, citation resolvability. Plus the decision
  policy (§13).
- `agent/decision.py`: the `DecisionEngine` selecting exactly one of the 6 states from validated
  evidence. Missing/conflicting/below-threshold → `AMBIGUOUS_REQUIRES_REVIEW`. Scheduled ≠ completed.
  **False-closure is impossible by construction.**
- `tests/fixtures/extracted/`: pre-extracted recommendation + candidate fixtures for A–H (no LLM).
- `tests/unit/`: cover deadline, alias, temporal, final-vs-draft, appointment-vs-completion, each state
  transition, task idempotency.
