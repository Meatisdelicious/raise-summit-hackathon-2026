# T30 — Agent orchestrator

The heart of the demo (spec §6). Inference **proposes** (plan, queries, tool selection, explanation);
the deterministic core **disposes** (validation, state). **The LLM never overrides a validator.**

## Do
- `planning.py`: turn a recommendation into an evidence-search plan (resolve deadline → search orders →
  appointments → results → external results → exceptions → validate).
- `orchestrator.py`: the plan→hunt→validate→decide→act loop — call tools by class, retry retrieval on
  insufficient/contradictory candidates (≤2/class), branch on rejection, enforce ≤12 steps (exceed →
  `AMBIGUOUS_REQUIRES_REVIEW`), invalid model output retried once, then draft the operational action via
  `create_task_draft`. Persist `agent_runs` + `tool_calls` with a correlation id.
- `events.py`: emit the exact SSE events from `docs/CONTRACTS.md` §6.
- Integration test (replay): a full run over signature case E branches on the predates-instruction
  rejection and lands `OPEN_OVERDUE`; case A lands `COMPLETED`.
